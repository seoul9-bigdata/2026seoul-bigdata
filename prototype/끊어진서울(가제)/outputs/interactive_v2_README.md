## 끊어진 서울 - V2 시각화 프로토타입

- 철도 따라 150m마다 샘플점 선택
- 그 점에서 철도와 직각 방향 250m씩 양쪽에 테스트 포인트 배치
- 각 테스트 포인트에서 가장 가까운 보행 네트워크 노드 찾기
- NetworkX Dijkstra로 두 노드 간 실제 보행 최단거리 계산
- 우회비율 = 실제 보행거리 / (250×2 = 500m)

## 개선점

- 현재 무의미하게 철도와 수직인 250m 지점에 점 찍는중
  - 물, 산 등 의미없는 곳에 마커가 찍혀버림
- 이거 데이터셋 뽑아서 유의미한 출발 / 도착지로 계산해야 함
- 다른 보행거리 계산 알고리즘 / api 지원하는지 확인해야 함
