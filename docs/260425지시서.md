### 김성령 — 기후 재난 + 팀 공통 엔진

클로드 요약: OSM Dijkstra 파이프라인·426개 동 손실률 실측·"영향 노인 수" 지표가 이미 완성되어 있으며, 배정 도메인(쉼터) 외에 Part A(공통 베이스라인)·Part B3(결빙·열선)까지 자발 구축함. 통계 검정(상관 r + p값)과 이론값 대조(53% vs 실측 54.7%)까지 수행한 유일한 파트. 이터레이션 기록(v1→v2 실패→v3)까지 투명하게 남김.

1. **경사도 가중치 실험**(선택): 시간 있으면 OSM elevation 또는 SRTM DEM으로 edge별 기울기 추출 → 노인 속도 감쇠 0.7~0.8 곱. 서울 구릉지(강북·관악) 체감 보정. 시간 없으면 공통 한계에 한 줄 추가.
2. **열선 매칭률**(22.6%): Geocoding API 이중 사용 또는 수동 사전 보완으로 개선. 현 README에 이미 명시되어 있으므로 공모전 심사용으로는 지금도 수용 가능.
   **안 바꿔도 되는 것**: 상관분석(r=0.020) 결과·영향 노인 수 지표·우선 개입 지도·5지점 다점 비교 등시선·쉼터 격차 역전(93.4% → 89.2%) 모두 그대로 발표 자산으로 이관.

### 이정태 — 의료

클로드 요약: 의료 도메인 분석으로 동 단위(425개) 손실률 코로플레스 + 청년×노인 산점도(이론선 포함) + 구별 3패널 비교 + 분포 히스토그램 완성. 산점도에 `y=x` 기준선과 `(0.65)²` 이론선을 같이 그린 것은 팀 전체에서 가장 지적인 시각화 중 하나. 좌표계 엄밀성(EPSG:5174 → 5179 약국 좌표 정확 변환)은 팀 내에서도 돋보임.

1. **거리 측정 방법론 전환** 현재 유클리드 직선거리 기반 계산을 OSM 기반으로. 유클리드가 실제 네트워크 거리보다 낙관적으로 추정하기 때문에, 재계산 후 **서울 평균 의료 손실률은 현 55.9%보다 올라갈 가능성**이 높음 — 이는 오히려 "문제가 더 심각하다"는 서사로 사용 가능.
2. **속도 파라미터 조정**: 현재 1.20 / 0.78 → 팀 표준 **1.28 / 0.88 / 0.70(필요하다면)**로 통일
3. **동 이름 매핑**: 현재 `dong_name = dong_code`로 임시 처리된 상태. 양석준의 `DONG_NAME_MAP` + BND_ADM_DONG_PG의 동명 필드를 결합해 실제 동 이름 표기. 발표 시 "강남구 12345" 가 아니라 "강남구 대치1동"으로 나와야 함.
4. **의료 TOP 5 확충 우선순위 시각화 추가**: 현재 4개 산출물 중 `04_distribution.html`(히스토그램)이 인사이트 전달력 약함. 이 슬롯을 **"의료 공백 영향 노인 수 TOP 5동"** 바차트 또는 지도로 교체 권장. 김성령의 07c 패턴 차용 가능.
5. **03 구별 바차트 개선**: 3패널(전체/병의원/약국) 각각 재정렬되어 있어 패널 간 비교가 어려움. **한쪽 기준(예: 전체)으로 순서 고정** 후 나머지 패널도 그 순서로 배치.
6. **참신한 시각화 검토**: 02 산점도의 이론선(0.65² 기울기)은 이미 충분히 참신. 추가 필요 없음. 다만 지도 33MB는 좌표 정밀도 축소로 경량화.

**안 바꿔도 되는 것**: 02 산점도의 구조(y=x + 이론선), 시설 3종 분리 분석 구조, 좌표계 변환 로직.

### 심재현 — 생활 인프라

클로드 요약: 별자리지도의 5막 내러티브(고무줄 지도 → 별자리 → 사라지는 그림자 → 0.70의 긴장감 → 구별 지수)는 팀 전체를 통틀어 **관객 임팩트가 가장 큰 결과물**. 핸드크래프트 HTML/CSS·Noto Serif KR·이중언어 섹션 타이틀 등 브랜딩도 최상위. 은행 점포 24년간 58.8% 감소 시계열은 팀 내 유일한 역사적 서사.

1. **[공간 해상도 관련 - 가능하다면]** 현재 **자치구 25개 단위**로 집계되어 있음. 팀 나머지(김성령·이정태·양석준 재작업 후)가 전부 행정동 426개로 수렴하는데 이 파트만 10배 거친 격자면 **통합 레이더 차트에서 축이 안 맞는다** 전통시장 195·슈퍼 3,000샘플 위치는 이미 실좌표라 재계산에 데이터 걸림돌 없음.
2. **속도 모델 정리**: 현재 1.28 / 1.12 / 0.88 + p15 0.70의 6값 구조. 팀 표준으로 맞춰 **1.28 / 0.88 / 0.78** 3-tier로 단순화. 별자리지도의 "0.70의 긴장감" 섹션에서 쓰는 0.70 m/s는 **0.78 m/s(하위 15%)로 수정**하면서 법적 기준 0.80과의 틈새 서사는 유지 가능. <mark>이라고 클로드가 써주긴 했는데, 재현님 하신거 보면 슬라이드 쭉 왔다갔다하는거도 굉장히 이쁜거같아서 괜찮아요 참고만 해주세요. 아니면 슬라이더 기능 유지하되 일반인 / 보행보조노인 / 보행보조노인(하위15%) 눈금 표시정도? 암튼 이건 맨 마지막에 발표자료 만들면서 합칠 때 수정해도 되긴 할듯</mark>
3. **SUP 샘플링 제거**: 현재 31,024개 중 3,000개만 샘플링해 사용 중. 샘플링 근거가 README에 없음. **31,024 전체로 재계산** 권장. 그래프 엔진이 사전 계산 캐시(REACH 테이블) 방식이면 성능 부담도 크지 않음.
4. **은행·주민센터 레이어 추가**: 별자리지도의 "사라지는 그림자" 섹션이 은행 점포 감소를 말하지만, 인터랙티브 대시보드의 필터에는 은행이 없고 전통시장·슈퍼만 있음. **서사와 구현 간 어긋남**. 은행·주민센터를 dashboard 시설 필터에 추가하면 D2 도메인이 브리프가 약속한 "쇼핑+금융" 두 축을 모두 커버.
5. **인프라 TOP 5 확충 우선순위 시각화 추가**: 현 `05 / 서울 25개 구 고령 보행권 지수`를 동 단위로 바꾸면서 **영향 노인 수 기반 TOP 5** 포맷 추가. 김성령의 07c 패턴 차용.
6. **v1(infra_dashboard_1.html) 아카이브**: v2와 98% 중복. 최종 배포 폴더에는 v2만. v1은 `legacy/`로 이동.
7. **README 파라미터 표기 통일**: 현재 "0.70 m/s"와 버튼 라벨 "보조기구 노인 0.88 m/s"가 섞여 있음. 팀 표준에 맞춰 정리.

**안 바꿔도 되는 것**: 5막 내러티브 구조, 별자리·고무줄·블랙홀·사라지는 그림자·0.70 긴장감 등 은유 네이밍, 핸드크래프트 HTML 디자인, 횡단보도 시뮬레이션, 은행 점포 시계열 데이터. **발표 오프닝·클로징은 거의 확정적으로 심재현 자산으로 채워진다.**

### 양석준 — 복지·녹지

클로드 요약: 6단계 파이프라인(analysis.py 545줄) + 단독 시각화 스크립트 3종 + 행정동 단위 정밀 분석(visualize_dong.py) + CSV 리포트 2종. 형식적 완성도는 팀 평균 이상. 특히 Nominatim 지오코딩 캐시와 `DONG_NAME_MAP` 정규화는 팀 공통 유틸로 쓸 가치가 있음. 복지 × 녹지 2D 사분면 산점도는 직관적 해석 가능.

1. **[최우선] 청년/노인 속도 구분 도입**: 현재 단일 400m/800m 버퍼로 운영 중이며, 어느 속도를 가정한 버퍼인지 코드/README에 명시되어 있지 않음. **팀 주제의 핵심인 "청년 vs 노인 격차"가 이 파트에만 없음**.
2. **[최우선] `analysis.py`의 구 centroid 근사 제거**: 현재 복지시설 모든 개체를 구 중심점 단일 좌표에 놓고 버퍼를 그리기 때문에, 400m/800m 원이 구 중심에 100% 중첩된다. 이 상태에서는 버퍼의 기하학적 의미가 성립하지 않음. **이미 `visualize_dong.py`에서 Nominatim 실지오코딩 결과를 `geocode_cache.json`에 저장해뒀으므로, 그 캐시를 analysis.py의 메인 파이프라인에 연결만 하면 된다.** 추가 API 호출 비용 0.
3. **[최우선] 복지시설 필터 재검토**: `welfare_df[welfare_df['자치구구분'] == '자치구']` 필터 결과로 강북구 3개·구로구 2개·강동구 5개가 나옴. 실제 서울 경로당은 자치구당 100~200개 이상. 이 필터가 **"자치구 직영 시설"만 걸러내는 것이라면 의도된 결과**지만, PR.md의 "경로당·노인복지관 목록"과 **라벨링 불일치**. 원본 CSV의 `자치구구분` 컬럼 가능한 값을 전부 확인 후, 전체 시설을 포함하도록 필터 재설계. 결과 숫자가 현재의 20~50배로 늘어나면 정상.
4. **Vulnerability Score 재설계**: 현 공식 `0.40×(1−복지지수) + 0.30×(1−녹지지수) + 0.30×(고령화율)` 에서 **고령화율 30% 가중을 제거**. 김성령의 실측 결과(고령화율 ↔ 손실률 `r=0.020, p=0.68`)가 무상관임을 보였으므로 이 항을 식에 넣으면 "고령화 높은 곳이 자동으로 취약 상위로 올라오는" 순환논리가 된다. 대체안: `영향 노인 수 = 손실률 × 65+ 인구` (김성령 지표) 차용.
5. **공간 단위 승격**: 현재 메인 결론(TOP5 리포트)은 25구 단위. `visualize_dong.py`로 이미 행정동 지도는 만들었으므로, **동 단위 Vulnerability + TOP 5동 리포트**로 승격 가능. 필요한 작업은 `master` 데이터프레임을 동 단위로 다시 groupby.
6. **복지·녹지 TOP 5 확충 우선순위 시각화**: 현 `top5_urgent_districts.csv`를 동 단위로 재계산한 뒤, 지도·바차트로 시각화. CSV만 있고 지도·차트가 없음.
7. **참신한 시각화**: 복지 × 녹지 2D 사분면 산점도(`welfare_green_scatter.png`)가 이미 나쁘지 않음. 유지하되 동 단위로 바꿔 점 개수를 25→426으로 늘리면 **클러스터 구조**가 드러날 가능성. 추가 아이디어 없으면 이걸로 충분.
   **안 바꿔도 되는 것**: Nominatim 캐시 로직, Spatial Join 코드 구조, `DONG_NAME_MAP` 내용, CSV 리포트 포맷, 사분면 산점도 시각화 아이디어. **구조는 살리되 핵심 기계(속도·좌표·필터·지표 공식)를 교체**하는 작업.
