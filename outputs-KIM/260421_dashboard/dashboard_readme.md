### 1. 보행 도달권 격차 지도
**파일**: `260418_submit/outputs/05_folium_map.html`  
**라이브러리**: Folium (Python → HTML)  
**작업일**: 260418

#### 구현 방법

```
OSM 보행 그래프(서울 전역, osmnx) 로드
→ 행정동 중심점에서 single_source_dijkstra
  · 일반인: 보행속도 1.28 m/s, 30분 = 2,304m cutoff
  · 노인:   보행속도 1.12 m/s, 30분 = 2,016m cutoff
→ 도달 노드 집합 → concave hull(alpha) → 폴리곤
→ Folium GeoJson 레이어로 두 폴리곤 겹쳐 표시
→ 행정동 경계 오버레이
```

#### 의미

같은 30분이어도 노인은 일반인보다 좁은 반경 안에 갇힌다. 지도 위에서 두 등시선의 면적 차이를 눈으로 보여 주는 가장 직관적인 출발점이다. "서울은 노인에게 더 작은 도시"라는 명제를 공간적으로 증명한다.

### 2. 보행권 손실률 지도
**파일**: `260418_submit/outputs/06_seoul_loss_map.html`  
**라이브러리**: Folium + Branca 코로플레스  
**작업일**: 260418

#### 구현 방법

```
손실률 = (일반인 30분 면적 − 노인 30분 면적) / 일반인 30분 면적

행정동별 손실률 계산 → 5분위 색상 매핑 (연두→빨강)
→ 행정동 폴리곤에 fill_color 적용 → Folium Choropleth
→ 상위 20개 동 별도 마커로 하이라이트
```

#### 의미

단순 격차가 아닌 **비율**로 측정함으로써, 도심(이동량 자체가 큰 곳)과 외곽(이동량이 작아 차이도 작게 보이는 곳)을 공정하게 비교한다. 손실률이 높은 동 = 노인의 이동 가능 범위가 상대적으로 가장 많이 줄어드는 곳. 구릉지(강북·관악·도봉)와 강변 단절 지역에서 두드러진다.

### 3. 보행권 우선 개입 지도
**파일**: `260418_submit/outputs/07c_priority_map.html`  
**라이브러리**: Plotly (버블 지도)  
**작업일**: 260418

#### 구현 방법

```
우선순위 스코어 = 손실률 × log(65세 이상 인구 + 1)

행정동별 스코어 계산
→ Plotly scatter_mapbox: x=경도, y=위도
  버블 크기 ∝ 스코어 / 색상 ∝ 손실률
→ 호버: 동명, 스코어, 손실률, 노인 인구 표시
```

#### 의미

손실률만 보면 "사람이 없는 산자락"이 최상위에 올 수 있다. 우선순위 스코어는 **격차 × 영향받는 노인 수**를 곱해, 정책 개입이 실제로 많은 사람에게 도움이 되는 동을 선별한다. "어디에 보행 인프라를 투자해야 최대 효과를 낼 수 있는가"에 직접 답한다.

### 4. 다점 비교 등시선
**파일**: `260420_submit/outputs/05_a1_dual_isochrone_260420.html`  
**라이브러리**: Leaflet (임베딩 방식)  
**작업일**: 260420

#### 구현 방법

```
5개 출발지 (탑골공원·미아역·신림동·삼성역·잠실새내) 선택
× 3 속도 그룹 (일반인 1.28 / 노인 1.12 / 보조장치 0.88 m/s)
× 3 시간대 (15 / 30 / 45분)
= 45개 등시선 미리 계산 → JSON 캐시

single_source_dijkstra → 도달 노드 → concave_hull(ratio=0.05)
→ 지점 선택 드롭다운 + 시간 슬라이더로 동적 레이어 교체
→ 서울 주요 랜드마크 도달 여부: 초록(전 그룹 도달) / 노랑(일반인만) / 회색(불가)
```

#### 의미

260418의 고정 1개 지점을 5개로 확장해 서울 각지의 격차를 비교한다. 강남(삼성역) 일반인 45분 26.5 km² vs 보조장치 노인 12.0 km²처럼 동일 지점·동일 시간에서도 속도 그룹에 따라 면적이 절반 이하로 줄어드는 현실을 슬라이더로 체험할 수 있다. 랜드마크 레이어를 통해 "노인이 30분 안에 갈 수 없는 병원이 어디인지"도 확인된다.

### 5. 폭염쉼터 노인 보행 접근성
**파일**: `260420_submit/outputs/06_b1_heat_shelter_260420.html`  
**라이브러리**: Leaflet  
**작업일**: 260420

#### 구현 방법

```
서울시 무더위쉼터 4,107개 → 그래프 노드 스냅 (nearest_nodes)
→ multi_source_dijkstra_path_length(G, all_shelter_nodes, cutoff=2304*1.05)
  단일 실행으로 모든 노드의 "가장 가까운 쉼터까지 최단 거리" 산출
→ 행정동 중심 노드의 거리값 추출
→ 5단계 접근성 분류:
    both_15(≤792m) · young_15(≤1152m) · both_30(≤1584m)
    young_30(≤2304m) · neither(>2304m)
→ 행정동 폴리곤 색상화 + 쉼터 마커 오버레이
→ 15분/30분 기준 토글 버튼
```

#### 의미

4,107개라는 높은 밀도 덕분에 서울 93.4%의 동에서 보행보조장치 노인도 15분 이내 폭염쉼터에 닿는다. 단, 나머지 6.6%(약 28개 동)는 사각지대. 이 동들이 어디인지 지도에서 직접 확인하는 것이 이 시각화의 핵심 가치다.

### 6. 한파쉼터 노인 보행 접근성
**파일**: `260420_submit/outputs/07_b2_cold_shelter_260420.html`  
**라이브러리**: Leaflet  
**작업일**: 260420

#### 구현 방법

```
서울시 한파쉼터 1,642개 (JSON 필드: lat / lot)
→ 동일 멀티소스 Dijkstra 파이프라인 (5번과 동일)
→ 결과: 380/426 동 = 89.2% 노인 15분 접근 가능
```

`lat`/`lot` 필드명 특이사항 주의 — 한파쉼터 JSON은 경도 필드명이 `lon`이 아닌 `lot`임.

#### 의미

**한파쉼터는 폭염쉼터보다 2.5배 적다.** 노인이 접근 어려운 동 비율이 7% → 11%로 커진다. 낙상·저체온증 위험이 가장 높은 겨울철에 오히려 접근 인프라가 더 부실하다는 역설이 핵심 메시지다. 두 지도(5번↔6번)를 같이 보면 이 격차 역전이 명확하게 드러난다.

### 7. 겨울 결빙 취약지점
**파일**: `260421_submit/outputs/10_b3_snow_icing_v3_260421.html`  
**라이브러리**: Leaflet  
**작업일**: 260421

#### 구현 방법

```
[기반 마스크 생성]
  서울 행정동 426개 unary_union  →  605.8 km² 기반
  OSM natural features 다운로드 (water 411개 + forest 393개 + wood 991개)
  → unary_union → 서울 내 173.9 km² 제거
  clean_seoul = 기반 − natural  →  431.9 km²

[결빙 취약 구역]
  제설함 10,437개 (EPSG:5186 mm단위 → WGS84 변환)
  → 100m / 200m 버퍼 unary_union
  icing = clean_seoul − covered  →  GeoJSON simplify(0.0005°) 직렬화

[열선 도로 라인]
  CSV 도로명 파싱 → 구 코드(adm_cd[:5]) 매핑 → GU_CODE_MAP
  → OSM 엣지 name 속성 매칭 (구 공간 경계 필터 적용)
  → 180/798 구간 매칭 → 9,212 엣지 → 주황색 라인 렌더링
```

**캐시**: `natural_features.json`, `icing_v3.json`, `heat_lines.json` — 재실행 시 계산 생략

#### 의미

"제설함이 없는 곳 = 제설 초동 대응 불가 = 결빙 지속 = 낙상 위험"의 논리로 도심 보행 위험 구역을 공간화한다. 한강·산지를 제거한 도심 기준으로 100m 반경에서 65%, 200m에서도 33%가 사각지대다. 열선(공식 결빙 위험 인정 구간) 레이어와 겹쳐보면 "알면서도 방치된 구간"을 식별할 수 있다.

## 한계 공통 사항

- **행정동 중심점 단일 출발**: 동 내 실제 주거 분포 미반영. 동 경계에 가까운 거주자의 실제 접근 거리와 다를 수 있음
- **평균 보행속도 적용**: 개인별 신체 능력 차이 미반영. 보행보조장치 노인 0.88 m/s는 집단 평균치
- **그래프 완전성**: OSM 기반으로 미등록 골목·사유지 통로 누락 가능. 실제 보행 경로보다 약간 과소 추정될 수 있음
