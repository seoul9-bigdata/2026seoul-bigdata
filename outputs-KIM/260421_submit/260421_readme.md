2. **결빙취약 폴리곤 오염**: 서울 전체 경계에서 커버리지만 빼는 방식 → 한강·산지 등 비보행 지역이 결빙 취약 구역으로 포함됨

## 개선 1 — 열선: 버블 집계 → OSM 도로 라인 시각화

### 기존 방식 (260420)

열선 CSV(`22_자치구별 도로열선 설치현황_2026.csv`)에는 구별 총 연장(m)만 있어서, 구 단위 버블 마커로 집계 표현에 그쳤다. "강북구에 열선이 많다"는 정보는 알 수 있었지만, 어느 도로인지는 알 수 없었다.

### 개선 방식 (260421)

CSV에는 도로명이 텍스트로 기재되어 있다 (`명륜길(명륜길 74 ~ 명륜9길 1)` 형식). 이를 파싱하여 OSM 보행 그래프 엣지의 `name` 속성과 매칭했다.

```
파이프라인:
  CSV 도로명 파싱 → 구명 추출 (adm_cd[:5] → GU_CODE_MAP)
  → 구 공간 경계로 OSM 엣지 필터
  → 엣지 name 속성 매칭
  → 매칭된 엣지를 GeoJSON 라인으로 직렬화
  → Leaflet L.geoJSON 주황색 선 렌더링
```

**결과**:
- 798개 구간 중 180개 매칭 (9,212개 OSM 엣지)
- 미매칭 원인: 도로 개명, 자치구 경계 인접 도로, 세부 도로 이름 OSM 미기재

**검증**: 주요 매칭 도로의 좌표를 직접 확인.
- 명륜길: lat 37.5877~37.5911 / lon 126.9928~126.9976 → 종로구 혜화동·명륜동 ✓
- 사직로1길: lat 37.5726~37.5748 / lon 126.9618~126.9633 → 종로구 행촌동 ✓

지도에서 도로 선을 클릭하면 도로명·자치구·설치 연장을 인포박스로 확인할 수 있다.

## 개선 2 — 결빙취약 폴리곤: 전체 서울 → 한강·산지 제외 도심 기준

### 기존 방식 (260420)

서울 경계 전체에서 제설함 커버리지를 빼는 방식이었다. 결과적으로 한강 수면, 관악산·북한산·수락산 등 산지, 공원 내 비보행 구역이 모두 "결빙 취약 구역"으로 표시됐다. 수치상으로도 비현실적이었다.

### v2 시도 (실패) — OSM 도로망 버퍼 마스크

OSM 보행 그래프 엣지를 18m 버퍼로 합산한 "보행 가능 도로망 영역"을 기반 마스크로 사용했다. 한강과 산지는 자연스럽게 제외됐지만, 도로 버퍼 사이 건물 블록 내부에 수천 개의 삼각형 구멍이 생겨 폴리곤이 시각적으로 매우 지저분해졌다.

### v3 해결 방식 (260421 최종)

```
기반 = unary_union(서울 426개 행정동 폴리곤)   → 605.8 km²
제거 = OSM water(자연수역 411개)
     + OSM forest(산림 393개)
     + OSM wood(자연식생 991개)
     → unary_union → 실제 서울에서 173.9 km² 제거

clean_seoul = 기반 − 제거  → 431.9 km²

icing_100 = clean_seoul − 제설함 100m 커버
icing_200 = clean_seoul − 제설함 200m 커버
```

행정동 경계는 원래 연속 폴리곤이므로 내부 구멍이 없다. 여기서 한강·산지만 잘라내면 건물 블록 구멍 없이 깨끗한 도심 기반이 완성된다.

**결과**:

| 제설함 반경 | 결빙취약 면적 | 도심 대비 비율 |
| ----------- | ------------- | -------------- |
| 100m 커버리지 기준 | 280.7 km² | **65.0%** |
| 200m 커버리지 기준 | 143.2 km² | **33.1%** |

기준 면적이 바뀌었기 때문에 260420의 수치(100m: 73.1%, 200m: 45.9%)와 직접 비교는 의미 없다. 중요한 변화는 "이제 한강·산이 결빙 취약 구역에서 빠졌다"는 것이다 — **실제 사람이 걷는 도심 공간만 기준으로 측정하게 됐다**.

## 주요 발견 종합

### B3 핵심 메시지 (260421 기준)

```
한강·산지를 걷어낸 서울 도심 431.9 km² 기준:
  제설함 100m 이내  →  도심의 35%만 커버
  제설함 200m 이내  →  도심의 67%만 커버

즉, 노인이 실제 걷는 서울 도심의 33~65%는
겨울철 제설 초동 대응 인프라 사각지대다.
```

**두 레이어를 겹쳐보면**:
- 열선 있음 + 제설함 미커버 → 결빙 위험을 알지만 사후 대응에만 의존하는 구간
- 열선 없음 + 제설함 미커버 → 행정 사각지대 (낙상 위험 가장 높음)

## 한계 및 유의사항

- **열선 매칭률 22.6%**: CSV의 도로명이 OSM `name` 속성과 완전히 일치하지 않아 약 78%의 구간이 미매칭. OSM에 이름이 없는 소규모 도로, 개명된 도로, 구 경계 인접 도로가 주요 원인
- **제설함 커버 = 제설 가능 전제**: 실제 제설은 구청 차량 제설도 있음. 본 분석은 사전 배치 인프라(제설함)의 공간 분포를 측정하며, 실제 제설 시행 여부와 다를 수 있음
- **결빙취약 폴리곤 단순화(simplify 0.0005°)**: 파일 용량 최적화를 위해 약 55m 이하 세부 형상 생략. 실제 결빙 위험 경계와 수십 미터 오차 가능
- **OSM natural features 완전성**: OSM에 등록되지 않은 소규모 수역·녹지는 여전히 결빙취약 구역에 포함될 수 있음
