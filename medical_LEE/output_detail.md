## 01 `01_medical_access.py` → outputs 01~04

### 핵심 로직

```
동 centroid → 반경 내 시설 수 카운트
  일반인: 1.28 m/s × 1800s × tobler(slope) → 동별 가변 반경
  노인:   0.88 m/s × 1800s × tobler(slope) → 동별 가변 반경
  손실률 = (1 - 노인 도달수 / 일반인 도달수) × 100
```

**경사 보정 (Tobler's Hiking Function):**

```
표고점(N3P_F002.shp) → 행정동 공간결합 → 동별 h_range = h_max - h_min
slope_est = h_range / (√동면적 × 1.13)   ← 원형 동 가정, 지름 근사
V(s) = 6 × exp(-3.5 × |s + 0.05|)        ← Tobler 함수
ratio = V(slope_est) / V(0)               ← 0 < ratio ≤ 1
일반인 반경 = 2304m × ratio
노인 반경   = 1584m × ratio               ← 동일한 ratio 적용
```

**설계 이유**

일반인과 노인 모두에게 같은 경사 패널티 비율을 적용합니다. 같은 비율을 곱하면 경사 효과가 분자·분모에서 상쇄되어, 손실률은 순수하게 **속도 차이(0.88 vs 1.28 m/s)에서 오는 접근성 격차**만 반영합니다. 노인에게만 경사를 적용하면 실제보다 손실이 과장되어 데이터 왜곡이 발생합니다.

검증 지표: 이론적 손실률 `1 - (0.88/1.28)² ≈ 52.7%` vs 실측 **51.7%** — 두 값이 근접하면 모델이 올바르게 동작하고 있다는 뜻입니다.

## 02 `02_temporal_access.py` → outputs 05~07

### 핵심 로직

```
노인(0.88 m/s) 반경 고정 1,584m 내 시설 중
진료시간(요일)S ~ 진료시간(요일)C 컬럼 파싱
→ 해당 시각에 실제로 열린 시설만 카운트

손실률 = (1 - 해당시간 열린 시설수 / 평일낮 기준) × 100
```

분석 시간대 4개:

| 슬롯 | 기준 |
|------|------|
| 평일 낮 | 월요일 14:00 (기준) |
| 평일 저녁 | 월요일 20:00 |
| 토요일 | 토요일 14:00 |
| 일요일 | 일요일 12:00 |

**설계 이유**

`01`이 공간적 접근성을 보여줬다면, `02`는 **시간적 접근성**을 추가합니다. 노인은 이동이 느릴 뿐만 아니라 진료 가능한 시간대도 제약이 많습니다. "같은 동이라도 시간에 따라 접근 가능한 시설이 얼마나 줄어드는가"를 수치로 표현합니다.

**결과**

| 시간대 | 평균 손실률 |
|--------|------------|
| 평일 저녁(20시) | **75.6%** |
| 토요일(14시) | **39.8%** |
| 일요일(12시) | **6.3%** |

일요일 손실률이 작은 이유: 약국이 일요일에도 상당수 운영하기 때문입니다. 가장 충격적인 수치는 평일 저녁 75.6%로, 퇴근 이후 시간대에는 의료 접근이 사실상 3/4 차단됩니다.

## 03 `03_trip_chaining.py` → outputs 08~09

### 핵심 로직

```
각 동 centroid(집)에서 scipy.spatial.cKDTree로:
  d1: 집 → 가장 가까운 병원
  d2: 그 병원 → 가장 가까운 약국
  d3: 그 약국 → 집 (귀가)
  총 동선 = d1 + d2 + d3

총 동선 > 1,584m  →  "30분 내 원스톱 불가"
병원 200m 이내 약국 존재  →  "원스톱 클러스터"
```

**설계 이유**

병원 한 번 가고 끝이 아닙니다. 처방전을 들고 약국까지 이동해야 합니다. 삼각 동선 전체가 노인의 30분 한계(1,584m) 안에 들어오는지 확인합니다. cKDTree를 사용한 이유는 426개 동 × 수만 개 시설 최근접 쿼리를 O(log n)으로 처리하기 위해서입니다. 단순 이중 루프 대비 수십 배 빠릅니다.

**결과**

| 지표 | 값 |
|------|-----|
| 30분 내 원스톱 불가 동 | **21개 (4.9%)** |
| 원스톱 클러스터 (병원+약국 200m 내) | **11,916개** |
| 원스톱 클러스터 30분 내 접근 가능 동 | **424개 (99.5%)** |
| 삼각 동선 평균 총 거리 | **576m** |

평균 576m는 노인 기준 30분의 약 36%에 해당합니다. 대부분의 동에서 병원→약국 원스톱이 가능하지만, 21개 동은 삼각 동선 자체가 노인 한계를 초과합니다.

## 04 `04_slope_correction.py` → outputs 10~11

### 핵심 로직

```
표고점(N3P_F002.shp, 76,580개) → 행정동 공간결합
동별: h_range = h_max - h_min
slope_est = h_range / (√area × 1.13)
V_adj = 0.88 × tobler(slope) / tobler(0)
dist_adj = V_adj × 1800
추가 손실 = (1 - dist_adj / 1584) × 100
```

**설계 이유**

`01`에서 경사 보정을 분석 로직에 통합했다면, `04`는 경사 그 자체를 독립 시각화합니다. "서울 어느 동이 가장 가파른가", "그 경사 때문에 노인이 실제로 얼마나 더 손해를 보는가"를 직접 보여줍니다. 지역별 지형 불평등을 별도 레이어로 드러냅니다.

**결과**

| 지표 | 값 |
|------|-----|
| 서울 평균 경사도 추정값 | **0.0695** |
| 경사 보정 후 노인 평균 반경 | **1,268 m** (평지 1,584m) |
| 경사로 인한 추가 손실 | **19.9%** |
| 가장 가파른 동 1위 | 노원구 중계4동 (slope=0.326) |
| 중계4동 노인 도달 거리 | **506 m** (평지 대비 -64%) |

경사가 가장 심한 동에서 노인의 실질 이동 거리는 평지의 1/3 수준으로 떨어집니다.

## 05 `05_elderly_population.py` → outputs 12~13

### 핵심 로직

```
구 단위 65세 이상 인구 비율 (주민등록인구 통계, 2025 4분기)
    ×
구 단위 의료 접근 손실률 (01 로직 동일하게 재계산, Tobler 동등 적용)

z-score 표준화:
  elderly_z = (elderly_pct - μ) / σ
  loss_z    = (loss_pct   - μ) / σ
  double_jeopardy_score = (elderly_z + loss_z) / 2

중위값 기준 4분면:
  고령↑ & 손실↑  →  이중 취약
  고령↑ & 손실↓  →  고령·양호
  고령↓ & 손실↑  →  시설 과소
  나머지          →  기타
```

**설계 이유**

손실률이 높다고 무조건 위험하지 않습니다. 노인 인구가 많은 곳에 의료 공백이 겹쳐야 실제 사회적 문제가 됩니다. z-score를 사용한 이유는 두 지표(%, %)가 단위는 같지만 분포 형태가 달라 직접 합산하면 편향이 생기기 때문입니다. 표준화하면 두 축이 동등한 가중치를 가집니다.

**결과**

| 지표 | 값 |
|------|-----|
| 서울 전체 65세 이상 비율 | **20.0%** |
| 이중 취약 구 수 | **7개** |
| 이중 취약 1위 | **강북구** (고령 26.0% / 손실 56.6%) |
| 이중 취약 2위 | **종로구** (고령 20.8% / 손실 62.8%) |
| 이중 취약 3위 | **도봉구** (고령 25.9% / 손실 49.6%) |

종로구는 고령화율이 중간이지만 손실률이 가장 높아 이중 취약에 포함됩니다. 반면 강남구는 손실률이 낮아 고령화율과 무관하게 안전지대입니다.

중위값 기준 4분면 구분선과 우상단 이중 취약 구역 음영 표시.
